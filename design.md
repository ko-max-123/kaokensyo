# 面接・会話練習向け 表情・視線クセ可視化Webサービス 設計書（MVP）

## 0. この設計書の位置づけ

この文書は、コーディングLLMにそのまま渡して実装を依頼するための設計書である。  
目的は、面接や会話練習時の「表情」「顔の向き」「視線の近似方向」「眉・目・口の動き」をブラウザ上で可視化し、自分のクセを把握できるWebサービスを作ることである。

本システムは以下の方針を前提とする。

- 独自AIモデルは作らない
- 学習処理はしない
- サーバーコストを極力かけない
- ブラウザ側で顔解析を行う
- 継続無料プランに寄せた構成にする
- 動画そのものは保存しない
- 保存するのは数値ログと集計結果のみ
- 医療診断・心理診断・採用判定は行わない

---

## 1. サービス概要

### 1-1. サービス名（仮）
Face Habit Viewer

### 1-2. 目的
ユーザーがカメラの前で話したときの表情や視線のクセを、感覚ではなく数値で確認できるようにする。

### 1-3. 解決したい課題
ユーザーは以下を把握しづらい。

- 口角が下がって見えていないか
- 顔が左右どちらかに偏っていないか
- 視線がカメラから外れやすくないか
- 眉を動かしすぎていないか
- まばたきが多すぎないか
- 緊張時に表情が固まっていないか

### 1-4. 想定利用シーン
- 面接練習
- 自己紹介練習
- プレゼン練習
- 会話トレーニング
- オンライン会議前のセルフチェック

## 2. スコープ

## 2-1. MVPで実装するもの
- ブラウザからカメラを起動できる
- スタートボタンで解析開始
- 停止ボタンで解析終了
- 顔ランドマークを取得できる
- 眉、目、口、口角の動きを数値化できる
- 顔の向き（左右・上下）を近似推定できる
- 視線の近似方向を推定できる
- 時系列ログを保持できる
- 停止後にグラフと要約を表示できる
- セッション結果をローカル保存できる
- CSV / JSON でダウンロードできる
- 最低限のサーバーAPIを持つ

## 2-2. MVPで実装しないもの

- 感情分類の高度推定
- 会員登録

## 3. 基本方針

## 3-1. アーキテクチャ方針
解析コストを抑えるため、顔解析はサーバーではなくブラウザ側で行う。  
サーバーは最小限とし、以下のみ担当する。

- `/api/health` の疎通確認
- 将来拡張用の受け皿
- 静的配信の補助

## 3-2. 推奨技術スタック
- フロントエンド: React + TypeScript + Vite
- UI: React + CSS Modules または Tailwind CSS
- チャート: Chart.js
- 顔解析: MediaPipe Face Landmarker
- ホスティング: Cloudflare Pages
- 軽量サーバーAPI: Cloudflare Workers
- ローカル保存: IndexedDB
- 状態管理: React Context または Zustand
- テスト: Vitest + React Testing Library
- E2E: Playwright

## 3-3. 保存方針
MVPでは動画本体は保存しない。  
保存対象は以下のみとする。

- セッションID
- 開始時刻
- 終了時刻
- サンプリング時系列データ
- 集計結果
- ユーザー設定値

## 4. ユーザー要件

## 4-1. ユーザーストーリー
- ユーザーとして、カメラを起動して表情解析を始めたい
- ユーザーとして、話している間の表情のクセを後から見返したい
- ユーザーとして、顔の向きや視線の偏りを知りたい
- ユーザーとして、結果をCSVで保存したい
- ユーザーとして、過去セッションと比較したい

## 4-2. 成功条件
ユーザーが練習終了後に、最低でも以下を確認できること。

- 顔が正面を向いていた割合
- 左右上下への顔向き傾向
- 視線がカメラ方向に近かった割合
- 口角上昇の平均
- 目の開きの平均
- まばたき回数
- 眉の上下動の大きさ
- セッション全体の傾向コメント

## 5. 機能要件

## 5-1. カメラ制御
### 要件
- カメラ利用許可を求める
- 許可後、映像をプレビュー表示する
- 未許可時はエラー表示する
- スタート押下で解析開始する
- 停止押下で解析停止する

### 入力
- ユーザーのカメラ映像

### 出力
- 画面上のプレビュー映像
- 解析中フラグ
- エラー文言

## 5-2. 顔検出・ランドマーク取得
### 要件
- 1名のみを対象とする
- 顔が見つからない場合はそのフレームを無効扱いにする
- 顔ランドマークを毎フレームまたは間引きサンプリングで取得する

### 備考
- 毎フレームの全記録は負荷が高い場合があるため、ログ保存は 5〜10fps 程度に間引いてよい
- 描画は requestAnimationFrame ベース、ログ保存は interval ベースでもよい

## 5-3. パーツ動き解析
### 対象部位
- 左右眉
- 左右目
- 口
- 口角
- 顔全体

### 算出したい指標
- 左眉の上下動
- 右眉の上下動
- 左目の開き
- 右目の開き
- 口の開き
- 口の横幅
- 左口角の上昇量
- 右口角の上昇量
- 顔の左右向き
- 顔の上下向き
- 視線の左右近似
- 視線の上下近似

## 5-4. 視線近似判定
### 方針
厳密なアイトラッキングではなく、以下の近似評価とする。

- 顔の向き
- 目周辺ランドマーク
- 虹彩または目の相対位置
- 正面基準との差分

### 出力例
- `camera_focus_score`: 0.0〜1.0
- `gaze_horizontal`: left / center / right
- `gaze_vertical`: up / center / down

## 5-5. 集計表示
### 表示内容
- 総計測時間
- 有効フレーム率
- 正面顔割合
- 左右向き割合
- 上下向き割合
- 視線中央割合
- 平均口角スコア
- 平均目開き
- まばたき回数
- 眉上下動の平均
- セッション中の最大変動値

## 5-6. グラフ表示
### グラフ候補
- 時系列折れ線グラフ
  - 顔左右向き
  - 顔上下向き
  - 視線中央スコア
  - 口角スコア
  - 目の開き
- 集計棒グラフ
  - 正面 / 左 / 右
  - 上 / 中 / 下
  - 視線 center 比率
- ヒートマップはMVP外

## 5-7. ローカル保存
### 要件
- セッション結果をブラウザ内に保存できる
- セッション履歴一覧を見られる
- 履歴削除できる

## 5-8. エクスポート
### 要件
- CSV ダウンロード
- JSON ダウンロード

### CSV列の例
- timestamp_ms
- face_detected
- face_yaw
- face_pitch
- gaze_horizontal_score
- gaze_vertical_score
- camera_focus_score
- left_eye_open
- right_eye_open
- mouth_open
- smile_score
- left_brow_raise
- right_brow_raise

## 6. 非機能要件

## 6-1. パフォーマンス
- 一般的なノートPCブラウザで動作すること
- 解析中もUIが極端に固まらないこと
- ログ保存は必要に応じて間引くこと
- 顔解析処理は重い場合、Web Worker の利用を検討すること

## 6-2. セキュリティ
- HTTPS 前提
- カメラ権限の明示
- 保存データはローカルのみ
- 動画そのものは保存しない
- API は個人情報を受け取らない

## 6-3. プライバシー
- 初期状態でクラウド送信しない
- セッションデータはブラウザ内にのみ保存
- ユーザーが明示的にダウンロードした場合のみ外部持ち出し

## 6-4. 可用性
- サーバーダウン時でもメイン解析機能は極力使えること
- API がなくても基本操作が可能であること

## 9. 指標定義

## 9-1. 顔向き
### 定義
- `faceYaw`: 左右向き
- `facePitch`: 上下向き
- `faceRoll`: 傾き

### 分類例
- yaw < -threshold: left
- -threshold <= yaw <= threshold: center
- yaw > threshold: right

## 9-2. 視線近似
### 定義
目・虹彩・顔向きを元に、カメラ方向に対する近似スコアを算出する。

### 出力
- 0.0〜1.0 の連続値
- center 判定用の閾値を持つ

## 9-3. 目の開き
### 定義
上下まぶたランドマーク間距離を正規化した値

### 用途
- 目が細い / 開いている傾向
- まばたき検出

## 9-4. まばたき
### 定義
目の開きが閾値以下になった区間を1回として数える  
連続フレームの重複カウントを防ぐデバウンス処理を入れる

## 9-5. 口の開き
### 定義
上下唇間距離を正規化した値

## 9-6. 笑顔スコア
### 定義
左右口角の上昇量、口幅、口開き補正を組み合わせて 0.0〜1.0 に正規化する  
MVPでは簡易式でよい

## 9-7. 眉の動き
### 定義
眉ランドマークの上下位置を基準値と比較し、上下動を数値化する

## 21. 実装時の注意事項

- 厳密な「視線追跡」と表現しないこと
- 「視線の近似評価」「カメラ方向への向きやすさ」と表現すること
- 診断や合否判定を示唆しないこと
- 顔が検出できないフレームを無理に補完しないこと
- 閾値は設定画面から変更可能にしておくこと
- 指標計算式は後から差し替えやすく分離すること
